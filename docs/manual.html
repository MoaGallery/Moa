How to use Moa

The key (and only, really) thing for getting to grips with Moa is to understand that it is tag (keyword) based. Every image should have tags attached to identify what it is an image of. Every gallery needs tags to tell it what it should contain. How complex you make your tag system is up to you. Images are not directly associated with galleries and can appear in multiple ones. Images without any tags will not be displayed in any gallery.

For example a gallery with tags of 'holiday' and 'Hawaii' will contain any image that has at least those two tags as well, even if they have others as well. This lets you tag images in lots of ways to appear in more than one gallery. An image in that example gallery could have tags of 'holiday', 'Hawaii', '2005', 'beach', 'dad' and 'family'. This lets you create another gallery called with the 'Family' tag and it will go in there as well, no need to upload the image twice.

Galleries can have sub-galleries inside them as well. These do not have to inherit the tags of it's parent although they will be filled-in automatically when you create one. By default any gallery with sub-galleries will not show any images inside, just the list of sub-galleries. This can be changed by editting the config.php file and changing $DISPLAY_PLAIN_SUBGALLERIES to true. As of version 1.0 this doesn't look very good but is planned to be worked on for version 1.1.

To add, edit or delete galleries and images use the links just below the header. You need to be logged in to do this. Galleries can be added from the main page or inside another gallery. Images can be added inside any gallery. Edit and delete can be done on any gallery or image. Deleting a gallery with sub-galleries will move those sub-galleries to where the parent was.

The admin button will let you do several things. You can create extra users or change your name/password. Extra users will all have access to the same galleries as you. The 'admin' tickbox against a user is whether to let them add/edit other users. You can add, edit and delete tags on the tag admin page. The last two pages are for finding any problems or lost images (not tagged).
